使用方法

日常开发流程

cd /Users/anglv/Nexus\ Chat

# 1. 本地开发 (isProduction = false，默认状态)
cd nexus-chat-app
flutter run

# 2. 修改代码后提交
git add .
git commit -m "feat: 描述你的改动"
git push

# 3. 部署到生产环境
./deploy.sh backend   # 部署后端
./deploy.sh app       # 构建生产 APK
./deploy.sh all       # 两者都执行

部署脚本命令

./deploy.sh help      # 查看帮助
./deploy.sh backend   # 部署后端到服务器
./deploy.sh app       # 构建生产 APK
./deploy.sh all       # 部署后端 + 构建 APK
./deploy.sh logs      # 查看服务器日志
./deploy.sh status    # 检查服务器状态

  ---
环境切换说明

| 环境     | isProduction | 说明                       |
  |----------|--------------|----------------------------|
| 本地开发 | false        | 连接本地后端 (localhost)   |
| 生产环境 | true         | 连接服务器 (8.130.161.255) |

部署脚本会自动处理环境切换 - 构建 APK 时自动切换到生产环境，构建完成后恢复为开发环境。
